OPI Documentation

Complete reference for the Omnikarai Package Index and the omnip CLI.

install

Install a package from OPI or from the current directory.

omnip install math_extra      # from OPI registry
omnip install .               # from current directory

Packages install to %USERPROFILE%\.omnikarai\modules\<name>\. Use in .ok files with:

use math_extra

uninstall

omnip uninstall math_extra

list

List all installed packages with versions and descriptions.

omnip list

info

Show details about a package (installed or from OPI).

omnip info math_extra

search

omnip search math
omnip search string

publish

Publish to OPI. Requires an API token (see API tokens).

set OPI_TOKEN=opi_your_token
omnip publish .

update

omnip update              # update all remote packages
omnip update math_extra   # update one package

init

Create an omnikarai.toml in the current directory.

omnip init

omnikarai.toml

[metadata]
name        = "my_package"
version     = "1.0.0"
description = "Does something useful"
author      = "Your Name"
license     = "MIT"
homepage    = "https://github.com/you/my_package"

[dependencies]
math = ">=1.0"

API Tokens

Create an API token in your Dashboard → API Tokens tab.

set OPI_TOKEN=opi_your_token_here
omnip publish .

Or pass via header when using the REST API directly:

Authorization: Bearer opi_your_token_here

REST API

Base URL: https://opi-nine.vercel.app

MethodEndpointDescription
GET/api/packagesList / search packages
GET/api/packages/:namePackage info + latest version
POST/api/packagesPublish a package (auth required)
DELETE/api/packages/:nameDelete package (owner only)
PATCH/api/packages/:nameYank a version (owner only)
GET/api/search?q=Search packages
GET/api/statsRegistry statistics
POST/api/auth/registerCreate account
POST/api/auth/loginSign in
GET/api/auth/meCurrent user info
GET/POST/DELETE/api/user/tokensManage API tokens
PATCH/api/user/profileUpdate profile

Example: publish via curl

curl -X POST https://opi-nine.vercel.app/api/packages \
  -H "Authorization: Bearer opi_your_token" \
  -H "Content-Type: application/json" \
  -d '{"name":"my_pkg","version":"1.0.0","description":"My package","license":"MIT","keywords":[],"dependencies":{}}'
