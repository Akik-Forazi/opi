Building omnip

Compile (GCC)

gcc -O2 -o omnip.exe omnip/src/omnip.c -lwinhttp -lkernel32

Compile (MSVC)

cl /O2 omnip.c winhttp.lib kernel32.lib /Fe:omnip.exe

Add omnip.exe to your PATH and you're ready.

Commands

omnip install <package>     # install from OPI
omnip install .              # install local module
omnip uninstall <package>   # remove package
omnip list                   # list installed packages
omnip info <package>         # show package details
omnip search <query>         # search OPI registry
omnip publish .              # publish to OPI
omnip update [package]       # update packages
omnip init                   # create omnikarai.toml
omnip version                # show version

API tokens

Create an account, then go to Dashboard → API Tokens → New token.

Set before publishing:

set OPI_TOKEN=opi_your_token_here

omnikarai.toml

Run omnip init to generate a template:

[metadata]
name        = "your_package"
version     = "1.0.0"
description = "What your package does"
author      = "Your Name"
license     = "MIT"

[dependencies]
# math_extra = ">=1.0"

Publishing

omnip publish .

omnip reads omnikarai.toml and sends metadata to OPI. Bump version for each new release.

Local database

Installed packages are stored at ~/.omnip/installed.json:

[{"name":"math_extra","version":"1.0.0","source":"opi-nine.vercel.app","install_path":"C:\\Users\\...\\math_extra"}]

Modules live at ~/.omnikarai/modules/<name>/
