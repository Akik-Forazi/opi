omnip Reference

omnip is the official package manager for the Omnikarai language. It installs packages from the OPI registry or from local directories.

omnip <command> [options]

install

Install a package from OPI or from the current directory.

omnip install math_extra # from OPI registry omnip install . # from current directory (needs omnikarai.toml)

Packages are installed to %USERPROFILE%\.omnikarai\modules\<name>\

After installing, use in your .ok file with:

use math_extra

uninstall

Remove an installed package.

omnip uninstall math_extra

list

Show all installed packages with their versions and descriptions.

omnip list

info

Show detailed information about an installed package.

omnip info math_extra

search

Search the OPI registry by name, description or keywords.

omnip search math omnip search string

publish

Publish your package to OPI. You must have an omnikarai.toml in the current directory.

omnip publish .

This reads your omnikarai.toml, packages all .ok files, and uploads to OPI.

update

Update installed packages to their latest versions.

omnip update # update all omnip update math_extra # update one

init

Create an omnikarai.toml file in the current directory to start a new package.

omnip init

omnikarai.toml

Every package needs an omnikarai.toml at its root.

[metadata] name = "my_package" version = "1.0.0" description = "A useful Omnikarai package" author = "Your Name" license = "MIT" homepage = "https://github.com/you/my_package" [dependencies] math = ">=1.0"

Creating a Package

A minimal package is just a directory with omnikarai.toml and one or more .ok files.

my_package/ omnikarai.toml main.ok # exported functions helpers.ok # internal helpers

To publish:

cd my_package omnip init # creates omnikarai.toml if missing # edit omnikarai.toml omnip publish . # uploads to OPI

API — GET /api/packages

Returns all packages. Supports ?q=query for search.

GET https://opi.vercel.app/api/packages GET https://opi.vercel.app/api/packages?q=math

Response:

{ "total": 3, "packages": [ { "name": "math_extra", "version": "1.0.0", ... } ] }

API — GET /api/packages/:name

GET https://opi.vercel.app/api/packages/math_extra

API — POST /api/packages

Publish a new package. Body must be JSON.

POST https://opi.vercel.app/api/packages Content-Type: application/json { "name": "my_package", "version": "1.0.0", "description": "Does something useful", "author": "Your Name", "license": "MIT", "keywords": ["util", "math"], "files": [] }
